# Topic Selection Rules

Use these rules when choosing one AI topic for a carousel.

## 1. Recency

- Prefer topics published or updated recently.
- For "today" or "latest" requests, verify the date from reliable current sources.
- If the topic is older but newly relevant, explain why it is being revisited.

## 2. Information Value

Good carousel topics usually include:

- A product behavior change.
- A model capability change that normal users can feel.
- A workflow change for creators, designers, product managers, or teams.
- A new market signal, policy shift, pricing change, or platform strategy.

Weak topics usually include:

- Pure funding news with no clear user meaning.
- Vague rumors without reliable source links.
- Benchmark-only updates that are hard to explain visually.

## 3. Discussion Value

Choose topics that can support a calm point of view:

- "This changes the user entry point."
- "This makes the tool more like an assistant than a button."
- "This matters less for the feature itself and more for the workflow behind it."

## 4. Reader Accessibility

Prefer topics that can be explained without dense jargon. If a technical term is necessary, define it in one short sentence.

## 5. Visual Fit

Good topics can be split into 5-7 pages:

- One cover idea.
- One misconception.
- One analogy.
- Two or three breakdown points.
- One impact page.
- One recap.

Avoid topics that require a long article to make sense.

## 6. Source Reliability

Prioritize:

- Official company announcements.
- Product release notes.
- Research papers or technical reports.
- Reputable media with clear attribution.
- First-party demos or documentation.

Avoid unsourced social posts unless they are only used as discussion context.
